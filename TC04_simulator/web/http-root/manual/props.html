Dewar Setup Properties

Persistant setup values are stored in the Dewar.props file. This is loaded during servlet initialization (i.e. when the server is started). If a change to the properties file is required, the server must be killed and restarted.

Here is an example file:

#Dewar parameters
#
heaters.name1=1: <sup>3</sup>He switch
heaters.name2=2: <sup>4</sup>He switch
heaters.name3=3: Ultra pump
heaters.name4=4: <sup>3</sup>He pump
heaters.name5=5: <sup>4</sup>He pump
heaters.send=true
diode.addresses=4,5,6
diode.banknames=A,B,C
diode.switches=3
diode.switchstate=false
grt.addresses=1,2,3
pic.addresses=0x50,0x51,0x52,0x53,0x54,0x55

heaters.name#
These are the names to use on the heater control page.
heaters.send#
This takes a true/false value, to set whether or not to send the heater voltages in the owwl data stream. The default is false.
diode.addresses
These are the addresses (just the 3 LSBs) of the diode DAS (ADC) chips. Each DAS handles a bank of 7 diodes. The full set is loaded by 4,5,6. If fewer banks are in use, miss out the unused addresses, e.g. 4,6.
diode.banknames
These are the names given to the diode banks, in the same order as diode.addresses.
diode.switches
These are the 2-bit addresses (0 to 3) of any ADG715 diode thermometer shorting switches. These are employed to turn off the diode thermometers, eliminating their heat input to cold stages.
diode.switchstate
This is the state (true or false for on or off) to leave the diode switches in after startup. You may want to set this to false if your application is sensitive to heat intput from the diodes.
grt.addresses
These are the addresses (just the 3 LSBs) of the GRT DAS (ADC) chips. Each DAS handles 2 GRTs. The full set is loaded by 1,2,3. If fewer banks are in use, miss out the unused addresses, e.g. 1,3.
pic.addresses
These are the (complete) addresses of the GRT PIC chips. Each PIC handles 1 GRT. The full set is loaded by 0x50,0x51,0x52,0x53,0x54,0x55. If fewer banks are in use, miss out the unused addresses, e.g. 0x50,0x51,0x54,0x55. Make sure there are two PIC address for each DAS address, and that they are listed in the same order, with the A channel and then the B channel for each DAS.
