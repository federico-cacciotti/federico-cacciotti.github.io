Temperature Calibration Management

The SNAP-based Dewar controller may use calibration data to output real temperature values from Diode and RTD (GRT) sensors. Suitable calibration files must be provided by the user, and assigned to hardware channels through the following process.

  1. The calibration data should be in the form of a text file, consisting of two columns of data: temperature and sensor value (voltage for diodes, resistance for RTDs). Values may be separated by commas, tabs or spaces.
  2. The calibration file(s) must be uploaded to the SNAP by FTP. Place them in /web/calib/
  3. Execute the calibration servlet 
  4. Under Import Calibration From Text File select a file to import, choose the sensor type and which channels it should be allocated to. Then click Import. After a delay the page will reload, with the newly imported calibration data listed alongside the sensor allocation.
  5. Repeat this for each new calibration file.
  6. If you made a mistake, or want to change an allocation, use the Update and Delete buttons.
  7. When all the correct allocations have be made, click the link to finalize. This will take you to a page under the main Dewar servlet, which will list all the calibration assignments after it was reloaded them. The Dewar servlet should now send temperature data through the Owwl channel, if enabled.
